# 토스 스마트 발송 — 기능성 알림 콘솔 등록 가이드

## 접속 경로

```
앱인토스 콘솔 (https://console-apps-in-toss.toss.im)
→ 워크스페이스 선택
→ estate-quick-sale 미니앱 선택
→ 좌측 메뉴 "스마트 발송"
→ "스마트 발송 시작하기"
```

## 캠페인 1: 키워드 급매 알림

### 기본 정보

| 항목 | 입력값 |
|------|--------|
| 캠페인 제목 | `관심단지 키워드 급매 알림` |
| 캠페인 유형 | **기능성 메시지** |
| 캠페인 기간 | 2026-03-03 ~ 2027-12-31 |

### 푸시 알림 소재

| 항목 | 제한 | 입력값 |
|------|------|--------|
| **푸시 제목** | 최대 20자 | `{{complexName}} 급매 나왔어요` |
| **푸시 본문** | 간결하게 | `토스에서 시세보다 저렴한 매물이 올라왔어요. {{price}}부터 확인해 보세요.` |

> - 알림(인박스) 제목은 미니앱 이름("부동산 급매 알리미")으로 자동 설정됨
> - `{{userName}}`은 토스가 자동 주입하므로 context에 넘기지 않아도 됨

### 랜딩 URL

```
/complex/{{complexId}}
```

콘솔 입력란 기준으로:
- 기능성 메시지는 API 발송이므로 콘솔에 랜딩 URL 입력란이 있으면 위 경로 입력
- 미니앱 내부 라우트: `/complex/:id` → 해당 단지 상세 페이지로 이동
- 푸시 클릭 시 토스 앱 → 미니앱 → `/complex/12345` 형태로 열림

### 템플릿 변수 (context)

| 변수명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| `complexName` | string | 단지명 | 래미안 퍼스티지 |
| `pyeongType` | string | 평형 (빈 문자열 가능) | 84A |
| `price` | string | 호가 (포맷팅 완료) | 9억 5,000만 |
| `userName` | - | 토스 자동 주입 (전달 불필요) | 김토스 |

### 저장 후 → "검토 요청" 클릭

## 푸시 문구 설계 의도

### 왜 이렇게 썼는가

토스 기능성 메시지 가이드라인:
- 광고/유도 표현 금지 ("놓치지 마세요", "지금 바로" 등 불가)
- 불안감 조성 금지 ("곧 사라져요" 등 불가)
- 서비스 이용에 필요한 **사실 전달**만 가능
- "토스" 멘션 권장

이 제약 안에서 클릭률을 높이려면:
1. **제목에서 단지명 + 핵심 사실** → 내 관심단지라는 걸 바로 인지
2. **본문에서 가격 정보** → 궁금증 유발 (가격이 보이면 안 눌러볼 수 없음)
3. **짧게** → 잘려서 보이면 클릭률 급감

### 실제 치환 예시

**키워드 급매:**
```
제목: 래미안 퍼스티지 급매 나왔어요        (14자)
본문: 토스에서 시세보다 저렴한 매물이 올라왔어요. 9억 5,000만부터 확인해 보세요.
```

**가격 급매:**
```
제목: 래미안 퍼스티지 가격 떨어졌어요       (16자)
본문: 토스에서 관심 단지 호가가 낮아진 매물이 감지됐어요. 8억 2,000만부터 확인해 보세요.
```

### 단지명이 길 경우 (20자 초과 가능성)

"래미안 블레스티지" (8자) + " 급매 나왔어요" (8자) = 16자 → OK
"e편한세상 시티 한강신도시 2단지" (16자) + " 급매 나왔어요" (8자) = 24자 → 초과

→ 토스 푸시에서 초과분은 자동 말줄임(...)으로 잘림. 앞부분 단지명이 보이므로 클릭에 영향 없음.

## A/B 테스트 관련

기능성 메시지에서는 **A/B 테스트 안 해도 됨**.

- 기능성 메시지는 API 발송 방식이라 **소재 자동 최적화가 적용되지 않음**
- A안만 작성하면 충분
- 광고성 메시지에서만 소재 기반 A/B 최적화가 동작함

## 검수 완료 후 할 일

### 1. templateSetCode 확인

캠페인 상세 화면에서 각 캠페인의 **templateSetCode** 값을 복사.

### 2. 서버 환경변수 설정

```bash
# /home/ubuntu/estate_quick_sale/.env 에 추가
TOSS_TPL_KEYWORD_BARGAIN=캠페인1의_templateSetCode
TOSS_TPL_PRICE_BARGAIN=캠페인2의_templateSetCode
```

### 3. PM2 재시작

```bash
pm2 restart estate-web
```

### 4. dry-run 테스트

```bash
cd /home/ubuntu/estate_quick_sale
node scripts/notify-bargains.mjs --dry-run
```

대상자/매물 매칭이 정상 출력되면 실제 발송:

```bash
node scripts/notify-bargains.mjs
```

## 코드 ↔ 콘솔 매핑

```
콘솔 templateSetCode ─── .env TOSS_TPL_KEYWORD_BARGAIN
                         .env TOSS_TPL_PRICE_BARGAIN
                              ↓
                    api/_lib/toss-message.js → getTemplateCode()
                              ↓
                    scripts/notify-bargains.mjs → sendSmartMessage(userKey, code, context)
                              ↓
                    api/_lib/toss-mtls.js → mTLS POST
                              ↓
          POST https://apps-in-toss-api.toss.im/api-partner/v1/apps-in-toss/messenger/send-message
          Header: x-toss-user-key = 유저의 toss_user_id
          Body: { templateSetCode, context: { complexName, pyeongType, price } }
```

## 광고성 캠페인 (신규 유입용)

광고성 캠페인은 **마케팅 수신 동의한 토스 사용자 전체**에게 발송 가능.
기능성과 달리 API 발송이 아니라 **콘솔에서 세그먼트 + 소재 설정** 후 자동 발송.
A/B 소재 최적화가 동작하므로 A안/B안 모두 작성 권장.

---

### 캠페인 3: 서비스 소개 — "이게 뭐지?" 유입용

| 항목 | 입력값 |
|------|--------|
| 캠페인 제목 | `부동산 급매 알리미 서비스 소개` |
| 캠페인 유형 | **광고성 메시지** |
| 타겟 세그먼트 | 전체 또는 "부동산 관심" 세그먼트 |
| 랜딩 URL | `/` (미니앱 홈) |
| 캠페인 기간 | 2주 단위 운영 후 성과 보고 갱신 |

**A안 — 궁금증 유발형:**

| 항목 | 입력값 |
|------|--------|
| 푸시 제목 (20자) | `토스에서 급매 아파트 찾기` |
| 푸시 본문 | `시세보다 저렴한 매물만 골라서 보여드려요. 서울 전역 실시간 감시 중.` |

**B안 — 숫자 임팩트형:**

| 항목 | 입력값 |
|------|--------|
| 푸시 제목 (20자) | `서울 아파트 급매, 몇 건일까` |
| 푸시 본문 | `토스에서 3만 건 매물 중 급매만 골라 볼 수 있어요.` |

### 광고성 문구 작성 원칙 (토스 가이드 기반)

| 규칙 | 적용 |
|------|------|
| "토스" 포함 필수 | 모든 본문에 "토스에서" 포함 |
| 처음 보는 사람도 이해 가능 | "급매 = 시세보다 저렴한 매물" 맥락 전달 |
| 짧게 | 제목 20자 이내, 본문 1~2문장 |
| 다음 화면 예측 가능 | "매물 확인", "급매 현황" 등 구체적 |
| 금지: 불안감 조성 | "놓치면 후회", "곧 사라져요" 사용 안 함 |
| 금지: 근거 없는 과장 | "최저가 보장" 같은 표현 안 함 |
| 금지: 이모지/느낌표 | 사용 안 함 |

---

### 광고성 캠페인 운영 팁

1. **A/B 소재 둘 다 작성** — 토스가 자동으로 성과 좋은 쪽을 더 많이 발송
2. **2주 단위로 소재 교체** — 같은 문구는 CTR이 점점 떨어짐
3. **세그먼트 분리** — "전체"와 "미접속 7일" 캠페인을 별도 운영
4. **랜딩 URL은 `/`** — 홈에서 TOP10 급매가 바로 보이므로 첫인상 강력
5. **성과 모니터링** — 콘솔에서 노출 수, 클릭 수, 클릭률 확인 가능

## 참고사항

- 검수 소요: **영업일 2~3일**
- 캠페인 활성화 후 테스트 발송이 먼저 진행됨:
  - 오전 5시 전 활성화 → 오전 8시 테스트 → 오후 3시 본 발송
  - 오후 12시 전 활성화 → 오후 3시 테스트 → 다음 날 오전 8시 본 발송
- 테스트 발송은 **최초 1회만** 진행
- 워크스페이스당 **10만 건/일** 발송 제한
- 기능성 메시지에 광고성 내용(혜택 안내, 리텐션 유도 등)이 포함되면 검수 반려됨
- 사용자가 알림 수신을 해제할 수 있는 기능 필요 → 우리 앱의 "설정 > 알림 설정" 페이지에서 토글로 제공 중
